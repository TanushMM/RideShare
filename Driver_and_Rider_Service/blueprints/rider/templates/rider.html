Rider API Documentation

Welcome to the documentation for the /rider endpoint. Below are the details of the available routes and how to use them.

Note: All endpoints require a valid JWT token to be included in the Authorization header as a Bearer token. Example: 

    Authorization: Bearer {your_jwt_token}

1. / (GET)

Base endpoint for the rider module.

Method: GET

Response: Renders the rider.html template. 

    {
        "message": "success",
        "contents": "This is the /rider endpoint"
    }

2. /riderData (GET)

Retrieves all rider data from the database.

Method: GET

Response: JSON containing a message and the contents of the rider table. 

    {
        "message": "success",
        "contents": [
            ["1", "John Doe", "john.doe@example.com"],
            ...
        ]
    }

3. /getRider (POST)

Retrieves the data of a specific rider based on the provided ID in the request body.

Method: POST

Request Body: JSON containing the rider ID.

    {
        "id": "1"
    }

Response: JSON containing a message and the contents of the rider record, or a failure message if no match is found. 

    {
        "message": "success",
        "contents": ["1", "John Doe", "john.doe@example.com"]
    }
    
    {
        "message": "fail",
        "contents": "No Match"
    }
    
    {
        "message": "fail",
        "contents": "Database Error - {error_message}"
    }

4. /addRider (POST)

Adds a new rider to the database using the details provided in the request body.

Method: POST

Request Body: JSON containing the rider details.

    {
        "name": "John Doe",
        "email": "john.doe@example.com"
    }

Response: JSON containing a success message if the rider is added successfully, or a failure message if an error occurs. 

    {
        "message": "success",
        "contents": null
    }
    
    {
        "message": "fail",
        "contents": "Database Error - {error_message}"
    }

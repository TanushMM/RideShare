Driver Cab API Documentation

Welcome to the documentation for the /driver/cab endpoint. Below are the details of the available routes and how to use them.

Note: All endpoints require a valid JWT token to be included in the Authorization header as a Bearer token. Example: 

    Authorization: Bearer {your_jwt_token}

1. / (GET)

Base endpoint for the driver cab module.

Method: GET

Response: Renders a simple HTML response indicating the base endpoint. 

    
Hello from /driver/cab/

2. /cabData (GET)

Retrieves all cab data from the database.

Method: GET

Response: JSON containing a message and the contents of the cabs table. 

    {
        "message": "success",
        "contents": [
            ["1", "AADHAR123", "TN10AXXXXX", "Sedan", "Honda City"],
            ...
        ]
    }

3. /getCab (POST)

Retrieves the data of a specific cab based on the provided ID in the request body.

Method: POST

Request Body: JSON containing the cab ID.

    {
        "id": "1"
    }

Response: JSON containing a message and the contents of the cab record, or a failure message if no match is found. 

    {
        "message": "success",
        "contents": ["1", "AADHAR123", "TN10AXXXXX", "Sedan", "Honda City"]
    }
    
    {
        "message": "fail",
        "contents": "No Match"
    }
    
    {
        "message": "fail",
        "contents": "Database Error - {error_message}"
    }

4. /addCab (POST)

Adds a new cab to the database using the details provided in the request body.

Method: POST

Request Body: JSON containing the cab details.

    {
        "_id": "2",
        "aadhar_id": "AADHAR456",
        "vehicle_number": "TN10BXXXX",
        "vehicle_type": "SUV",
        "vehicle_model": "Toyota Fortuner"
    }

Response: JSON containing a success message if the cab is added successfully, or a failure message if an error occurs. 

    {
        "message": "success",
        "contents": null
    }
    
    {
        "message": "fail",
        "contents": "Database Error - {error_message}"
    }

API Documentation - /user Endpoint

GET /user

This endpoint serves the main HTML page for user operations. It renders the `user.html` template and does not return any JSON data.

Request

Method: GET

URL: http://your-server-ip:5050/user/

Response

The response is an HTML page rendered from the `user.html` template.

GET /user/userData

This endpoint retrieves all user data from the database and returns it in JSON format.

Request

Method: GET

URL: http://your-server-ip:5050/user/userData

Headers:

Authorization: Bearer <JWT_TOKEN>

Response

A successful response returns a JSON object containing a message and the user data.

{ "message": "success", "contents": [ { "_id": "60d21b4667d0d8992e610c85", "name": "John Doe", "email": "john.doe@example.com" // Other fields... } ] }
POST /user/getUser

This endpoint retrieves a specific user's data based on the provided ID.

Request

Method: POST

URL: http://your-server-ip:5050/user/getUser

Headers:

Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>

Body:

{ "_id": "60d21b4667d0d8992e610c85" }

Response

A successful response returns a JSON object containing the user's data.

{ "message": "success", "contents": { "_id": "60d21b4667d0d8992e610c85", "name": "John Doe", "email": "john.doe@example.com" // Other fields... } }
POST /user/addUser

This endpoint adds a new user to the database.

Request

Method: POST

URL: http://your-server-ip:5050/user/addUser

Headers:

Content-Type: application/json
Authorization: Bearer <JWT_TOKEN>

Body:

{ "_id": "60d21b4667d0d8992e610c85", "name": "John Doe", "email": "john.doe@example.com" // Other fields... }

Response

A successful response returns a confirmation message.

{ "message": "Admin/User added successfully" }
